# ЦЦР — бэкенд + админ-панель

Принимает заявки с контактной формы сайта в PostgreSQL (DigitalOcean) и даёт
админ-панель для управления клиентами (статус, комментарий, оценка).

## Состав
- `server.js` — Express-сервер (API + раздаёт админку)
- `db.js` — пул подключений к PostgreSQL
- `schema.sql` / `init-db.js` — таблица `leads`
- `public/admin.html` — админ-панель (синие тона)
- `.env.example` — пример конфигурации

## Шаги запуска

1. Установить зависимости:
   ```
   npm install
   ```

2. Скопировать конфиг и вписать пароль БД:
   ```
   cp .env.example .env
   ```
   В `.env` заполни `PGPASSWORD` (пароль от DigitalOcean) и поменяй `JWT_SECRET`
   на длинную случайную строку. Логин/пароль админа — `ADMIN_USERNAME` /
   `ADMIN_PASSWORD` (по умолчанию admin / admin — обязательно смени на проде).

3. Создать таблицу в базе:
   ```
   npm run init-db
   ```

4. Запустить сервер:
   ```
   npm start
   ```
   - Сайт-форма шлёт заявки на `POST /api/leads`
   - Админка: http://localhost:3000/admin.html (логин admin / admin)

## Подключение фронта (сайта) к бэкенду
В `script.js` сайта есть строка:
```js
const DDC_API_BASE = '';
```
- Пусто (`''`) — если сайт раздаётся тем же сервером, что и API.
- Иначе укажи адрес бэкенда, напр. `http://localhost:3000` (для локалки) или
  `https://api.tvoy-domen.kz` (прод). И добавь этот origin сайта в `CORS_ORIGIN` в `.env`.

## API
| Метод | Путь | Доступ | Назначение |
|------|------|--------|------------|
| POST | `/api/leads` | публичный | приём заявки с формы |
| POST | `/api/login` | публичный | вход админа → cookie-сессия |
| POST | `/api/logout` | — | выход |
| GET | `/api/me` | админ | проверка сессии |
| GET | `/api/leads` | админ | список клиентов (фильтр `?status=&q=`) |
| GET | `/api/stats` | админ | счётчики по статусам |
| PATCH | `/api/leads/:id` | админ | смена `status` / `admin_comment` / `rating` |

## Статусы клиента
`new` (новый) · `in_progress` (в процессе) · `on_hold` (отложен) ·
`served` (обслужен) · `rejected` (отказ)

## Безопасность (важно)
- Пароль БД и JWT-секрет хранятся только в `.env` (не коммить его в git).
- Пароль админа задаётся на сервере, на клиент не передаётся.
- На проде поставь HTTPS и `NODE_ENV=production` (тогда cookie станет `secure`).
- Логин/пароль admin/admin — только для старта, поменяй.
